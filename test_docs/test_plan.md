# Тест-план проекта TaskFlow

## 1. Цель тестирования
Проверка функциональности, безопасности и надежности системы управления задачами TaskFlow, а также корректности математических расчетов статистики.

## 2. Область тестирования
- Регистрация и аутентификация пользователей
- CRUD операции с задачами
- Статистика и расчеты (процент выполнения, средний приоритет, workload score)
- Изоляция данных между пользователями
- Валидация входных данных
- UI интерфейс пользователя

## 3. Виды тестирования

| Вид тестирования | Описание |
|------------------|----------|
| Функциональное | Проверка соответствия системы функциональным требованиям |
| Негативное | Проверка обработки некорректных входных данных |
| Тестирование граничных значений | Проверка границ допустимых значений (длина строк, диапазоны чисел) |
| Интеграционное | Проверка взаимодействия компонентов системы |
| Контрактное | Проверка соответствия API-ответов ожидаемой схеме |
| UI-тестирование | Проверка пользовательского интерфейса (Playwright) |
| Регрессионное | Проверка, что новые изменения не сломали существующую функциональность |

## 4. Тестовая среда

| Компонент | Характеристики |
|-----------|----------------|
| **Бэкенд** | Python 3.13, FastAPI, SQLite, локальный хост `http://127.0.0.1:8000` |
| **Фронтенд** | Статический HTTP-сервер (Python `http.server`) на порту 5500 |
| **ОС** | Windows 11 |
| **Браузер для UI-тестов** | Chromium (Playwright) |
| **Инструменты тестирования** | pytest, pytest-cov, requests, playwright |

## 5. Критерии начала тестирования
- Код собран и запускается без ошибок
- Бэкенд доступен на `http://127.0.0.1:8000`
- База данных инициализирована
- Фронтенд доступен на `http://localhost:5500`
- Базовые smoke-тесты проходят

## 6. Критерии окончания тестирования
- 100% запланированных тест-кейсов выполнены
- Критические дефекты (Severity: Critical, Major) исправлены
- Процент успешных автоматических тестов не менее 95%
- Покрытие кода тестами не менее 80%

## 7. Риски

| Риск | Вероятность | Меры по снижению |
|------|-------------|------------------|
| Нестабильность тестовой базы данных | Низкая | Использование отдельной тестовой БД с автоматическим сбросом после каждого теста |
| Зависимость от внешних библиотек | Средняя | Фиксация версий в `requirements.txt`, регулярное обновление |
| Сложность настройки Playwright | Низкая | Использование встроенных инструментов playwright для установки браузеров |
| CORS-настройки при UI-тестах | Низкая | Предварительная настройка CORS в бэкенде на `allow_origins=["*"]` |
